# AI 활용 수준 간이테스트 플로우

## 1. 플로우 개요

- 목표: 3문항 안에 사용자의 AI 활용 수준을 빠르게 드러내고, 결과를 `상위 x.x%`로 보여준다.
- 핵심 경험:
  - 짧다
  - 실무형이다
  - 결과가 다 다르게 나온다
- 구현 전제:
  - `바이브코딩으로 만든 정적 HTML 페이지`
  - 업로드 후 바로 배포 가능한 형태

## 2. 전체 여정

1. 랜딩 유입
2. 테스트 소개 확인
3. Part1 응답
4. Part2 응답
5. Part3 응답
6. 결과 계산
7. 상위 퍼센타일 노출
8. 해석 및 CTA 확인

## 3. 랜딩 화면

헤드라인은 호기심을 만들고, 바로 `점수+퍼센타일` 기대를 심어야 한다.

### 권장 카피

- 헤드라인:
  `나는 내 AI 스킬 수준은 상위 몇 %일까?`
- 서브카피:
  `단 3문항으로 확인하는 AI 활용 수준 테스트`
- 보조카피:
  `프롬프트 설계, 검증 사고, 실무 적용력까지 점수로 보여드립니다.`

## 4. 구현 및 배포 플로우

이 테스트는 초기에 다음 방식으로 구현된다.

1. 바이브코딩으로 HTML 프로토타입 제작
2. 정적 파일 구조로 마무리
3. HTML 업로드
4. 즉시 배포
5. 이후 필요 시 데이터 저장 기능 추가

즉, 제품 플로우뿐 아니라 제작 플로우도 `정적 HTML 선배포`를 전제로 한다.

## 5. 시작 전 안내

사용자에게 아래 내용을 짧게 보여준다.

- 총 3문항
- 약 5분 소요
- 결과는 `점수 + 상위 퍼센타일`로 제공
- 누구나 서로 다른 `상위 x.x%` 결과를 받음

## 6. Part1 플로우

- 목적: 사용자가 AI 답변을 얼마나 비판적으로 읽는지 본다.
- 형식: 객관식 1문항
- UX:
  - 보기 4개
  - 바로 다음으로 이동

## 7. Part2 플로우

- 목적: 편향, 설명 책임, 인간 검토 필요성을 분석할 수 있는지 본다.
- 형식: 짧은 분석형 서술 1문항
- UX:
  - 사례 제시
  - 답변 박스 제공
  - 권장 글자 수 안내

권장 문구:

- `문제 3가지와 개선방안 2가지를 적어보세요.`

## 8. Part3 플로우

- 목적: 실제 업무 상황에서 AI에게 어떤 식으로 지시하고, 어떤 검증 루프를 설계하는지 본다.
- 형식: 실전형 1문항
- UX:
  - 간단한 데이터/상황 제시
  - `1차 프롬프트`
  - `2차 검증 질문`
  - `3차 개선 질문`
  순서로 쓰게 유도

## 9. 결과 계산 플로우

1. Part1 점수 계산
2. Part2 루브릭 점수 계산
3. Part3 루브릭 점수 계산
4. 총점 100점 환산
5. ranking score 생성
6. 전체 응답자 분포와 비교
7. 상위 퍼센타일 계산
8. 결과 카피 매핑

## 10. 왜 binary가 아닌가

이 테스트는 `상위 7% 맞음/아님`으로 끝나면 재미도 없고 설득력도 약하다.

그래서 결과는 이렇게 설계한다.

- 모든 응답자는 자신만의 `상위 x.x%`를 받는다.
- 같은 총점이어도 실전형 문항을 더 잘한 사람이 더 높은 순위를 받는다.
- 결과가 촘촘하게 갈려서 비교 욕구가 생긴다.

## 11. 결과 화면 구조

### 1번 블록

- 총점
- 상위 퍼센타일
- 한 줄 진단

예시:

- `87점`
- `상위 11.8%`
- `AI를 단순 도구가 아니라 검증 가능한 업무 파트너로 쓰는 편입니다.`

### 2번 블록

- Part1 점수
- Part2 점수
- Part3 점수

### 3번 블록

- 강점 2개
- 취약점 2개
- 다음 행동 3개

## 12. 퍼센타일 노출 규칙

- 결과는 항상 `상위 x.x%` 형식으로 노출한다.
- `상위 7%`는 특정 사용자에게만 강조 배지처럼 추가로 붙는다.

예시:

- `상위 4.9%`
  배지: `TOP 7%`
- `상위 18.6%`
  배지 없음
- `상위 52.3%`
  배지 없음

## 13. 결과 카피 분기

### 상위 7% 이내

- `AI를 꽤 잘 쓰는 수준이 아니라, 구조적으로 잘 쓰는 상위권입니다.`

### 상위 8~20%

- `활용 수준이 높습니다. 다만 검증 루프를 더 정교하게 만들면 상위권으로 바로 올라갈 수 있습니다.`

### 상위 21~45%

- `기본 활용은 익숙하지만, 아직 프롬프트 구조화와 리스크 인식에서 차이가 납니다.`

### 상위 46~70%

- `AI를 쓰고는 있지만 결과를 관리하는 방식은 아직 감각에 의존하는 편입니다.`

### 상위 71% 이하

- `사용 경험은 있으나 실무형 활용 습관은 아직 초입 단계입니다.`

## 14. 리드 수집 위치

간이테스트이므로 결과 직전 리드 수집은 무겁게 느껴질 수 있다.

권장 방식은 두 가지다.

1. 결과 먼저 보여주고 상세 리포트 저장용 이메일 받기
2. 결과 상단 일부 공개 후 상세 해설 보기 전에 이메일 받기

현재 목적이 공유성과 완주율이면 `결과 먼저`가 더 적합하다.

## 15. 배포 관점 UX 메모

- 정적 HTML로 먼저 배포할 것이므로 첫 버전은 빠르게 열려야 한다.
- 3문항 모두 한 화면 흐름 안에서 자연스럽게 전환되는 구조가 적합하다.
- 퍼센타일 계산과 결과 렌더링은 프론트엔드에서 즉시 끝나야 한다.
- 업로드 후 바로 배포하는 구조이므로 의존성 없는 구현이 가장 유리하다.

## 16. 공유 유도 포인트

`상위 x.x%` 구조는 공유에 유리하다.

권장 공유 문구 예시:

- `나는 AI 활용 수준 상위 12.4%가 나왔어요`
- `3문항 테스트 해봤는데 생각보다 빡세네요`
- `AI를 쓰는 것과 잘 쓰는 건 다르다는 게 점수로 보임`

## 17. 운영상 체크포인트

- 3문항이어도 쉬워 보이면 안 된다.
- Part2와 Part3가 실제 실력 차이를 만들어내는 핵심이다.
- 퍼센타일이 너무 뭉치지 않게 내부 점수는 소수점 단위로 계산한다.
- 결과는 반드시 `모든 사용자에게 다르게 보이는 느낌`을 줘야 한다.
- 구현은 `HTML 업로드 후 바로 배포`를 전제로 하므로, 초기 버전은 정적 환경에서도 100% 동작해야 한다.
