# AI 활용 수준 간이테스트 기술명세서

## 1. 문서 개요

- 프로젝트명: `AI 활용 수준 간이테스트`
- 테스트 타이틀: `나는 내 AI 스킬 수준은 상위 몇 %일까?`
- 메인 카피: `내 AI 활용 수준을 점수와 퍼센타일로 증명하세요.`
- 목적: 긴 시험이 아니라 `3문항만으로도 실력 차이가 드러나는` 마케팅용 진단 테스트를 설계한다.

## 2. 핵심 방향

- 이번 버전은 정식 APEX 시험 전체를 보여주는 것이 아니다.
- 대신 APEX 레포의 `Part1`, `Part2`, `Part3` 성격을 각각 1문항씩 뽑아 `간이테스트`로 재구성한다.
- 결과는 `상위 7%입니다 / 아닙니다`의 이분법이 아니라, 모든 응답자가 `상위 x.x%` 형태의 서로 다른 결과를 받도록 설계한다.
- 구현은 `바이브코딩 기반 정적 HTML`을 전제로 한다.
- 즉, 초기에 앱이나 복잡한 백엔드보다 `업로드 가능한 HTML 파일` 형태로 먼저 완성하고 바로 배포 가능한 구조를 우선한다.

## 3. 테스트 구조

- 총 문항 수: `3문항`
- 구성:
  - `Part1` 객관식 1문항
  - `Part2` 분석형 1문항
  - `Part3` 실전형 1문항
- 예상 소요 시간: `4~6분`
- 결과 방식:
  - 총점 `100점`
  - 상위 퍼센타일 `상위 x.x%`
  - 세부 역량 해석

## 4. 왜 3문항 구조인가

이 테스트는 길게 평가하기보다 짧게도 수준 차이가 드러나야 한다. 그래서 각 문항 역할을 분명히 나눈다.

1. `Part1`
   AI를 쓸 때 기본적으로 무엇을 의심하고 검증해야 하는지 아는가
2. `Part2`
   AI 활용의 위험과 편향을 분석하고 인간의 책임을 설명할 수 있는가
3. `Part3`
   실제 업무 상황에서 프롬프트, 보안, 검증, 개선 루프를 함께 설계할 수 있는가

## 5. 문항 선정 원칙

- 모두 APEX 레포에 이미 존재하는 문항 성격에서 가져온다.
- 다만 정식 시험 그대로 복제하지 않고, 간이테스트용으로 길이와 난도를 조정한다.
- 세 문항만 풀어도 아래 4가지는 반드시 드러나게 한다.
  - 프롬프트 설계
  - 검증 습관
  - 보안/윤리 인식
  - 업무 적용력

## 6. 점수 체계

- 총점: `100점`
- Part별 가중치:
  - `Part1 20점`
  - `Part2 35점`
  - `Part3 45점`

가중치를 이렇게 두는 이유는 다음과 같다.

- Part1은 지식 확인 성격이 강하므로 비중을 낮춘다.
- Part2는 판단력과 분석력을 본다.
- Part3는 실제 활용 실력을 가장 잘 드러내므로 비중을 가장 높인다.

## 7. 세부 채점 구조

### Part1

- 정답형 객관식
- 점수:
  - 정답 `20점`
  - 오답 `0점`

### Part2

- 분석형 서술 문항
- 5개 기준 × 7점 = `35점`
- 채점 기준 예시:
  - 편향 식별
  - 설명 책임 인식
  - 인간 최종결정 원칙
  - 법적/윤리적 위험 인식
  - 구체적 개선안 제시

### Part3

- 실전형 프롬프트/검증 설계 문항
- 5개 기준 × 9점 = `45점`
- 채점 기준 예시:
  - 역할과 맥락 설정
  - 보안/익명화 조건 반영
  - 출력 형식과 분석 범위 지정
  - 근거 vs 추정 구분 요구
  - 2차/3차 검증 및 개선 루프 설계

## 8. 퍼센타일 산출 원칙

이 테스트의 핵심은 `모든 사람이 서로 다른 상위 몇 %` 결과를 받게 하는 것이다.

그래서 결과는 구간형이 아니라 `연속형 percentile`로 계산한다.

### 기본 원칙

- 결과는 `상위 7%`, `상위 18.4%`, `상위 42.7%`처럼 나온다.
- 정수 퍼센트가 아니라 `소수점 1자리`까지 보여준다.
- 비교 기준은 `간이테스트 전체 응답자 풀`이다.

## 9. 퍼센타일 계산 방식

### 1차 점수

- `display_score = Part1 + Part2 + Part3`
- 범위: `0.0 ~ 100.0`

### 2차 랭킹 점수

동점자를 최대한 줄이기 위해 퍼센타일 계산용 내부 점수를 별도로 둔다.

- `ranking_score = display_score`
- `+ (part3_score / 1000)`
- `+ (part2_score / 10000)`
- `+ (verification_subscore / 100000)`

이렇게 하면 표시 점수는 깔끔하게 유지하면서도, 같은 82점 안에서도 실제로는 순서가 갈린다.

### 3차 퍼센타일 변환

- 전체 응답자 `ranking_score` 분포에서 누적 백분위를 계산한다.
- 노출 형식은 다음과 같다.
  - `당신은 상위 12.4%입니다`
  - `당신은 상위 37.8%입니다`
  - `당신은 상위 71.2%입니다`

## 10. 데이터가 적을 때의 처리

초기에는 실응답 데이터가 적을 수 있으므로 2단계로 운영한다.

1. `베타 단계`
   내부 기준 분포표를 사용해 임시 퍼센타일 산출
2. `정식 오픈 이후`
   실응답자 누적 데이터를 기준으로 퍼센타일 재계산

권장 기준:

- 실응답 `300명 미만`: 초기 벤치마크 사용
- 실응답 `300명 이상`: 실제 응답 분포 기반 전환

## 11. 결과 메시지 체계

`상위 7% 여부`는 강조 포인트일 뿐, 결과는 모든 사용자에게 다르게 준다.

### 예시

- `상위 4.8%`
  프롬프트 설계와 검증 사고가 모두 강한 편
- `상위 13.6%`
  실무 활용력이 높지만 보안/윤리 영역은 더 다듬을 여지 있음
- `상위 31.2%`
  AI를 꽤 활용하지만 구조화와 검증 습관에서 격차 발생
- `상위 58.9%`
  사용 경험은 있으나 아직 검색형 사용 비중이 높음
- `상위 82.3%`
  AI를 써보긴 했지만 실전 활용 체계는 아직 약함

## 12. 결과 화면 필수 요소

- 총점
- 상위 퍼센타일
- 한 줄 진단
- 3개 파트별 점수
- 잘한 점 2개
- 보완할 점 2개
- 다음 행동 제안 3개

## 13. 구현 및 배포 전제

- 1차 구현 형식:
  - `정적 HTML/CSS/JavaScript`
- 제작 방식:
  - `바이브코딩`
- 배포 방식:
  - HTML 파일 업로드 후 즉시 배포 가능한 구조

즉, 이 테스트는 처음부터 아래 전제를 가진다.

1. 별도 설치 없이 브라우저에서 바로 열릴 것
2. `index.html` 중심의 정적 파일만으로 동작할 것
3. 점수 계산과 결과 표시가 프론트엔드에서 우선 완결될 것
4. 초기 버전은 HTML 업로드만으로도 공개 가능한 상태일 것

## 14. 역량 해석 축

3문항이지만 결과 해석은 아래 4축으로 요약 가능하다.

1. `검증 감각`
2. `리스크 인식`
3. `프롬프트 구조화`
4. `실무 적용력`

## 15. 운영상 주의사항

- 이 테스트는 `3문항 간이 진단`이지 공식 자격시험이 아니다.
- 따라서 `합격/불합격`보다 `상위 몇 %` 경험이 중심이 되어야 한다.
- 결과가 너무 뭉치지 않도록 반드시 `소수점 점수 + tie-break ranking score` 구조를 쓴다.
- 같은 총점이어도 Part3가 강한 사용자가 더 높은 퍼센타일을 받게 설계한다.
- 구현은 `HTML 업로드 후 바로 배포`를 전제로 하므로, 초기 버전은 정적 환경에서도 100% 동작해야 한다.
